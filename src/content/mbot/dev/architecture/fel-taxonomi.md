# Fel-taxonomi

Denna sida är avsedd att innehålla fel-taxonomin som styr hur klient och server ska agera vid olika typer av fel.

Lägg in taxonomin här när den är finaliserad i separat tråd.

Rekommendation:
- definiera felklasser
- definiera defaultåtgärd per klass
- koppla varje klass till testfall i testsuite
- definiera vad som loggas och vad som larmas

Obs: Fel-taxonomi påverkar även hur multi-server och idempotens implementeras.
